4-7 A sample web app

We’ll end this chapter with the implementation of a classic web app. The app folder is /chap4-7. Copy it to a local folder, and install dependencies with the command npm install. Then run the app with the command node app.js.

The app has the following views. These views contain buttons or links to initiate actions through controllers. E.g. the view GET /list.html has a link to delete the todos.

  • GET /list.html - lists all tasks in a <table>
  • GET /edit.html?id=x - a web form to show existing value of the todo with id ‘x’, and the user can edit the value
  • GET /add.html - a web form to input a new todo

The sample app has the following controllers.

  • POST /add - adds a new todo. Request body contains urlencoded name-value pairs of the new todo
  • POST /update - updates an existing todo. Request body contains urlencoded name-value pairs of the updated todo
  • POST /delete - deletes an existing todo. Request body contains the id of the todo to delete. Urlencoded.

The app implements Post-Redirect-Get patterns. The controllers above redirects to the view GET /list.html.
