4-4 Routing in web app

This lab first covers the concept of a Node app and dependency management in package.json. Next it starts the discussion on the web app framework Express.js.

Node app and package.json

In this lab, we’ll use the Express.js framework and some other modules to build web app server. As an Node application makes use of more NPM modules, it becomes difficult to install the modules manually with the command npm install module-name. Instead, you can describe the dependencies of a Node app in a JSON txt file called package.json.

A Node app contains 1 or more JavaScript file (*.js) together with modules and other assets (e.g. HTML files, images). All these file are put in a folder. A special file package.json describes the app. It defines the name and version, which identifies the app in the www.npmjs.com.

{
  "name": "comp312-examples",
  "description": "Examples for Express.js in comp312",
  "version": "0.0.1",
  "private": true,
  "dependencies": {
    "express": "^4.10"
  }
}

The config file above indicates that our app depends on the ‘express’ module. It requires a version compatible with 4.10. For more detail about the version specification, refer to the online document for Semvar.

After listing all the required modules for the current app in package.json, you can install all the dependencies with the command npm install. If later, you need to add a dependency (e.g. request module), run the following command.

npm install request --save

This command installs the latest version of specified module. In addition, it updates package.json and adds the module to the dependencies. Try the command and check the change in package.json.

Express.js

Express.js (website) is a web framework to simplify development of Web applications.

  • Middleware – chainable processing of requests and responses
  • Routing – different request processing based on URL patterns
  • Template – generate HTML output from variables

Basic structure of an express app.

var express = require('express');
// create an Express application, which will handle HTTP requests
var app = express();

// a route in the app, which handles GET request for the path '/'
app.get('/', function(req, res) {
  res.send('hello world');
});

// more routes ...

// start listening at TCP port 3000
app.listen(3000);

Routes

When a web server receives an HTTP request, it has to decide how to handle the request, and return the result as an HTTP response. Such decision usually depends on the HTTP method (either GET or POST in most cases) and the path in the URL (e.g. “/about.html”). The combination of HTTP method and the URL path is usually referred to as an endpoint of the web service.

In an Express app, we define a route to describe how to process HTTP requests at an endpoint. The general syntax is app.get(path, callback) and app.post(path, callback). The callback can take 2 or more parameters. Often, we only use the first two parameters called req and res. req refers to the incoming HTTP request, and res can be used to build the HTTP response.

// app is an Express app

app.get('/about.html', (req, res) => {
  // When the Express app receives a GET request
  // for the path '/about.html', it returns a text response
  res.send('This is a simple example of route')
});

Some web apps include parameters in the URL path. For example, the URL https://www.google.com/maps/place/Macao+Polytechnic+Institute/ refers to the place ‘Macao Polytechnic Institute’ in the web app Google Maps. You can easily extract these parameters in Express routes.

Consider the sample code below, which shows a web app to check lecture hours. When the app receives a GET request for the path ‘/lecture/comp312’, it returns a response of its lecture hours. The parameters are available in the object req.params. (The full source is app1.js).

// retrieve the lecture time
app.get('/lecture/:code', (req, res) => {
  // in a real app, we'd query a database ...
  if (req.params.code=='comp312') {
    res.send('Tue, Thu: 10:00-11:30am')
  } else if (req.params.code=='comp311') {
    // ...
  }
})

The above only describes the basics of routing in Express. Refer to the online guide for more possibilities.

The order of routes is significant! An Express app checks the routes in the order they are defined. When a route matches the HTTP methods and URL path, the Express app executes its callback function to handle the request. The callback usually returns an HTTP response with res.send() or similar methods. When a response is returned, the Express app will stop checking the remaining routes.

Making responses

You can use the res object in the route callback to build a response. (online reference) The res object has several common methods:

  • res.send(body) sends a response with the specified body. The body is usually a string, but it can also be JavaScript objects or others. The method determines the suitable MIME type.
  • res.sendFile(path) returns a file as response.
  • res.status(code) sets the HTTP response code. This function is chainable.
  • res.redirect(path) redirects to the given URL.

Check the source code of app2.js for demonstration of how to build responses with these methods.
