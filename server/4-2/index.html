4-2 NPM modules

In this lab, we will

  • install module with npm install
  • demonstrate how send HTTP requests using the request modules.
  • cover the concept of a Node app and dependency management in package.json

Installing Modules

Node.js has a built-in module called http and https for making HTTP clients and servers. They are efficient implementation of the HTTP protocol. However, the modules are low-level and not convenient to use. Therefore, the Node community has developed several modules for writing HTTP clients and servers. These modules are published in https://www.npmjs.com/. A popular choice for HTTP clients is the request module. (online reference).

Using Request to GET an URL

The following example fetches the current weather report from http://rss.smg.gov.mo/e_ActualWeather_rss.xml.

// p301.js
var request = require('request');

var url = 'http://rss.smg.gov.mo/e_ActualWeather_rss.xml';
request.get(url, (err, res, body) => {
  console.log('STATUS: ' + res.statusCode);
  console.log('HEADERS: ');
  console.dir(res.headers);
  console.log(body);
});

In order to run this program, you have to put the program in a project folder. Change the current directory to the project folder, then install the module with the command npm install request. (Note: the installation is also done once.) You’ll see that the command npm installs the module in a local folder called node_modules.

npm install request

The callback for request.get takes three parameters. The first is the error object. The second is an object of class http.IncomingMessage (online reference). It represents the HTTP response message. You can use it to check HTTP status code and headers. The last parameter is the message body of the HTTP response.

Options object pattern

The next example downloads an image from IPM website and saves it.

The JavaScript programming language does not support named parameters. Some APIs make use of a pattern caled options object pattern to pass named parameters to a function. For example, if you need to give more information about the HTTP request in addition to URL, you can group these information in an option object and pass it as the first parameter of reqeust.get(). Refer to online reference for all available options.

// p302.js
// - Download and save an image file from www.ipm.edu.mo

var fs = require('fs');
var request = require('request');

// options object pattern
// use a JS object to group named parameters
// encoding = null is required to get binary data
var options = {
  url: 'http://www.ipm.edu.mo/cntfiles/upload/images/common/campus/campusmap_btn_chi_un_img.jpg',
  encoding: null
};

request.get(options, (err, res, body) => {
  if (err) throw err;
  if (res.statusCode!==200) {
    console.log('HTTP status not ok. code = ', res.statusCode);
    return;
  }
  console.log('HTTP request is successful. Saving file...');
  fs.writeFile('campus.jpg', body, (err) => {
    if (err) throw err;
  });
});

Making POST request

In the next example, we want to download the class time table from the IPM web app at https://wapps.ipm.edu.mo/siweb/time_prog.asp. The web app uses HTTPS (instead of plain HTTP) and the POST method to send query to the server. The query data can be sent using the form property in the options object in request.post.

// p303.js
// - saves the response of a POST request to the file timetab.html

var request = require('request');
var fs = require('fs');

var form = {
  p_escl_cod: 5, /* ... */
  p_year_sem: '2015/2016-1',
  p_sp_year: 3,
  p_class_code: '31121'
};

var options = {
  url: 'https://wapps.ipm.edu.mo/siweb/time_prog.asp',
  form: form
};

request.post(options, (err, res, body) => {
  if (err) throw err;
  console.log('HTTP status: ' + res.statusCode);
  // write the HTML response body to an HTML file
  fs.writeFile('timetab.html', body, 'utf8', (err)=> {} );
});

Node app and package.json

As an Node application makes use of more NPM modules, it becomes difficult to install the modules manually with the command npm install module-name. Instead, you can describe the dependencies of a Node app in a JSON txt file called package.json.

You can create a package.json file for a Node app by running this command in the project folder.

npm init

A Node app contains 1 or more JavaScript file (*.js) together with modules and other assets (e.g. HTML files, images). All these file are put in a folder. A special file package.json describes the app. It defines the name and version, which identifies the app in the www.npmjs.com.

{
  "name": "comp312-examples",
  "description": "Examples of web client",
  "version": "0.0.1",
  "private": true,
  "dependencies": {
    "request": "^2.75"
  }
}

The config file above indicates that our app depends on the ‘request’ module. It requires a version compatible with 2.75. For more detail about the version specification, refer to the online document for Semver.

After listing all the required modules for the current app in package.json, you can install all the dependencies with the command npm install. If later, you need to add a dependency (e.g. cheerio module), run the following command.

npm install cheerio --save

This command installs the latest version of specified module. In addition, it updates package.json and adds the module to the dependencies. Try the command and check the change in package.json.
