3-8 Page load and others

Writing plan:

  • page load: two events: DOM ready, page load done.
  • most resources are loaded in parallel to DOM building
  • exception: <script>
  • error in the middle of script element
  • long load time of <script src=..>
  • DOM tree is incomplete when the browser is blocked by script
  • so … defer loading of script at the end of body
  • or … $(function() { … })

Asynchronous programming

  • callback, 2 kinds: synchronous and asynchronous.
  • .. [].map(x=>x+1) vs. $().on(‘click’, (e)=>{ }) or setTimeOut()
  • JS is single threaded
  • event loop runs while DOM is loading? i believe NO
  • experiment: setTimeOut(), then alert() to block the thread
  • dispatch event handler 1 by 1

Event delegation

  • $().on( ) does not bind to new elements
  • bubbling
  • delegation. event.target, event.currentTarget
  • handle in ancestor

  • stop propagation

  • register at an ancestor

  • add selector in $().on()

  • works for newly added elements

  • .. but cannot add instance specific data in .on()

  • .. solution: attach data to the instance with $().data()

  • make an example of dynamically added elements

Basic operation

We call the constructor of class Chart to create a chart. The constructor accepts two parameters: the first is a canvas element, and the second is a JSON data object (a JavaScript object) which has at least three properties: chart type, data to be plotted, and options. Try the demo p371.html.

<!-- a canvas where we'll draw a bar chart -->
