3-5 Form controls

This lab covers jQuery methods and events related to form control.

Text input

The basic control for entering text is text input boxes. Use the method $().val() to get / set the value of a text input box.

<input type='text' name='email' value='initial value'/>

<script>
  // use attribute 'name' to select the input control
  // use .val() to read the text value
  var s = $('input[name="email"]').val();

  // change the text value of the text input box
  $('input[name="email"]').val('peter@ipm.edu.mo');
</script>

When the input box gains the input focus, keyboard input changes its value. Try the demo p341.html and examine when the events focusin, focusout, change and click occur.

Selection

There are several form controls that allow the user to select one out of several options. The HTML <select> element is the easiest to work with using jQuery. The following code snippet from the demo p342.html demonstrates how to set/change the selection. The demo also shows some events that the <select> element fires.

<select name="program">
  <option value='computing'>Computing</option>
  <option value='social'>Social works</option>
  <option value='pa'>Public admin</option>
</select>

<script>
  // use attribute 'name' to select the select control
  // use .val() to read the value of the selected option
  var s = $('select[name="program"]').val();
  // s is either 'computing', 'social' or 'pa'

  // change the selected option
  $('select[name="program"]').val('computing');
</script>

Another common control to capture user selection is radio button groups. They are more tricky to work with because the radio buttons in a group share the same name. To find out which button is checked, use the pseudo class $('input[ ]:checked'). It is also tricky to check an option in a radio button group: you have to put the value inside an array. Try the demo p343.html.

<input type="radio" name="IQ" value="low"/>&lt;80
<input type="radio" name="IQ" value="normal"/>80-120
<input type="radio" name="IQ" value="high"/>&gt;120

<script>
  // find out which radio button is selected
  var s = $('input[name=IQ]:checked').val();

  // to select an option. notice that you need to put the value in array
  $("input[name=IQ]").val(['high']);
</script>

A special case of selection is to ask a yes/no question. Usually, you would use a checkbox to ask a yes/no question. Although you can still use selector and methods similar to those of radio button groups to handle checkbox, it is usually easier to use the DOM property checked directly.

Real? <input type="checkbox" name="real" value='yes'/>

<script>
  // to find out whether the checkbox is checked
  var checked = $('input[name="real"]').prop('checked');
  // true or false

  // to check the checkbox
  $('input[name="real"]').prop('checked', true);

  // to uncheck the checkbox
  $('input[name="real"]').prop('checked', false);
</script>

The above form controls to make selection may trigger several events: click, change, focusin and focusout. The most commonly used is perhaps change, which fires when the selection changes. It fires before the control loses focus. (change event in text input box fires after the control loses focus.) The following code snippet comes from the demo p344.html.

<p>Which OS are you running?
  <input type="radio" name="os" value="windows"> Windows
  <input type="radio" name="os" value="macos"> macOS
  <input type="radio" name="os" value="other"> Other:
  <input type="text" name="otheros" value=""/>
</p>
<script>
$('input[name="os"]').on('change', (event)=>{
  // the radio button that the user checks
  if ($(event.target).val()==='other') {
    $('input[name="otheros"]').prop('disabled', false);
  } else {
    // disable the input text box, and clear its content
    $('input[name="otheros"]').prop('disabled', true).val('');
  }
})
</script>
